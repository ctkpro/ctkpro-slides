# CTK Pro 公司介紹投影片製作計畫

本文件整理新版 `ctkpro-intro` 投影片的製作方向。目標是替換目前偏向 BNI / 政府補助情境的舊版內容，改成給新認識的企業客戶使用的公司介紹簡報。

## 目標定位

新版簡報的主軸：

> CTK Pro 是協助企業把網站規劃、建置、上線與長期維護穩當處理好的技術夥伴。

這份簡報不主打工具清單，也不把 AI 放成核心服務，除非後續有可公開、可驗證的實際案例。對第一次認識 CTK Pro 的客戶，最重要的是建立三個印象：

- 我們懂企業網站建置。
- 我們能把網站穩定上線。
- 我們能在上線後持續維護與擴充。

## 版本策略

不再拆長版 / 短版，也不做四份獨立的深淺色版本。

最終只保留一份 `ctkpro-intro/index.html`，內建深色 / 淺色切換：

- 預設使用淺色模式，因為投影機環境最穩。
- 提供畫面上的 theme toggle。
- 支援鍵盤快捷鍵，例如 `T` 切換深色 / 淺色。
- 使用 CSS variables 控制整份 deck 的色彩，不複製內容。
- 可用 `localStorage` 記住上次選擇。
- 淺色模式使用彩色 logo，深色模式使用白色 logo。

## 設計方向

延續本 repo 既有簡報風格：

- 乾淨、簡潔、明亮。
- 投影機友善，避免過暗、過細、過小的文字。
- 使用 `yilan-bids/index.html` 的 1280 x 720 stage 架構作為骨架。
- 保留頁碼、進度條、左右 1/6 點擊翻頁、鍵盤翻頁、全螢幕等互動。
- 加入 4:3 safe area，確保低解析度投影機也能閱讀。

## 色彩規劃

色彩以 CTK Pro 官網與現有 logo 為準，避免沿用舊版簡報的藍色作為主色。

建議色彩 token：

```css
:root {
  --brand:#e11f27;
  --brand-700:#ad0204;
  --ink:#221012;
  --muted:#414143;
  --line:#E1E1E8;
  --bg:#FDFDFF;
  --paper:#FFFFFF;
}

[data-theme="dark"] {
  --brand:#e11f27;
  --brand-700:#ff8787;
  --ink:#FFFFFF;
  --muted:#CDCDCD;
  --line:#2E2E2E;
  --bg:#1C1C1C;
  --paper:#222222;
}
```

使用原則：

- 紅色只作為重點、頁碼、線條、章節標記與 CTA，不大面積鋪滿。
- 淺色版以暖白與清楚留白為主。
- 深色版避免做成高反差刺眼黑底，使用接近官網的深灰背景。
- 圖表與案例頁如需第二色相，再另行補一個輔助色，不預先擴張色票。

## 頁面大綱

### 1. 封面

主題：

> CTK Pro 竑盛科技

副標：

> 企業網站建置與長期維運夥伴

目的：建立品牌與服務定位，不放過多說明。

### 2. Index

目錄建議：

- 我們是誰
- 企業網站常見問題
- 我們怎麼協助
- 核心服務
- 案例分享
- Why Us
- 聯絡方式

目的：讓第一次見面的客戶知道簡報會如何展開。

### 3. 我們是誰

核心訊息：

> 我們專注在企業網站建置、上線營運與長期維護，協助客戶把網站做成可持續使用的數位資產。

可提及：

- 企業網站建置
- WordPress / CMS 客製
- 主機與上線設定
- 維護、備份、安全更新
- 後續功能擴充

### 4. 企業網站常見問題

建議用四格問題呈現：

- 網站不好改，內容更新要一直找工程師。
- 網站速度慢，手機體驗不佳。
- 上線後沒人維護，更新與備份不清楚。
- 每次新增需求都像重新做一個專案。

目的：讓客戶先感覺「這些問題我們懂」。

### 5. 我們的核心承諾

核心訊息：

> 我們不只做一個能看的網站，而是做一個能被管理、能穩定上線、能長期擴充的網站。

建議用三句收斂：

- 規劃清楚：架構、內容、動線先整理。
- 建置穩定：前台、後台、速度、安全一起考慮。
- 維護有人：上線後持續管理、備份、更新與調整。

### 6. 核心服務 1：網站規劃與資訊架構

重點：

- 網站架構規劃
- 頁面與內容盤點
- 使用者動線
- SEO 基礎結構
- 需求與範圍整理

目的：說明我們不是直接開工刻頁面，而是先協助客戶把網站該說什麼、怎麼說、怎麼被找到整理清楚。

### 7. 核心服務 2：網站設計、建置與後台管理

重點：

- RWD 響應式設計
- 前端頁面實作
- CMS / 後台管理
- 表單、內容模組、基本互動
- 效能與基礎安全設定

目的：把「網站做出來」講成客戶聽得懂的交付內容，而不是列技術名詞。

### 8. 核心服務 3：上線、主機、資安與維護

重點：

- 網站上線設定
- 主機與網域
- 備份與還原
- 版本更新與安全維護
- 後續內容調整與功能擴充

目的：強調網站不是交付就結束，CTK Pro 可以協助後續營運與維護。

## 案例頁內容原則

案例頁只放已確認可公開、可展示、可被追問的內容。

如果案例尚未補齊，先保留模板與 placeholder，不臆造客戶、數據、截圖或成果。

建議每個案例只回答三件事：

1. 客戶原本卡在哪裡？
2. CTK Pro 做了什麼？
3. 最後產生什麼可理解的成果？

## 技術與工具呈現原則

不做單純工具清單頁。

WordPress、Notion、Cursor、Claude、Gemini 等工具不作為主要賣點。若需要提及，必須轉譯成客戶價值：

- WordPress：成熟 CMS，讓客戶可以管理內容。
- Cloudways / 主機：穩定上線、備份與效能管理。
- 設計與開發工具：縮短溝通與製作時間，但不需要在主簡報中列名。

客戶真正需要聽到的是：

- 網站能不能準時上線。
- 上線後誰維護。
- 內容能不能自己改。
- 未來能不能擴充。
- 出問題時有沒有處理流程。

## 投影機與尺寸策略

維持單一 HTML deck，不依解析度拆版本。

實作時要檢查：

- 16:9：1280 x 720、1920 x 1080
- 4:3：1024 x 768、800 x 600

設計限制：

- 避免小字表格。
- 每頁只放一個主要訊息。
- 案例頁使用固定欄位與短句，不塞長段文字。
- 重要內容放在 4:3 safe area 內。
- 頁碼、切換按鈕與導覽不得遮住內容。

## 實作步驟

1. 以 `yilan-bids/index.html` 的 stage 與導覽邏輯作為骨架。
2. 重構 `ctkpro-intro/index.html`，移除舊版 BNI / 政府補助導向內容。
3. 建立 CTK Pro 品牌色 CSS variables。
4. 加入深淺色切換按鈕與快捷鍵。
5. 補上白色 logo 資產供深色模式使用。
6. 依本文件建立 13 頁投影片。
7. 第 9、10、11 頁先做案例模板，等待後續內容補齊。
8. 用 Playwright 或 `scripts/shoot.mjs` 檢查 16:9 與 4:3 視覺狀態。
9. 更新 root `index.html` 的公司介紹卡片文案，對齊新版定位。
